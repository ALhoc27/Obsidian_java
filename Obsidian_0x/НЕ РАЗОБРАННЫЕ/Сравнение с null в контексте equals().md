Метод `equals()` в Java является ключевым компонентом сравнения объектов, и важной его особенностью является поведение при сравнении объекта с `null`.

> [!NOTE] **Сравнение с `null`**:
> - Любой объект `x`, кроме `null`, при вызове `x.equals(null)` должен возвращать **`false`**.
> - Нельзя вызывать метод `equals()` у объекта, который равен `null` (это вызовет `NullPointerException`). 
>```java
>x.equals(null) // false
>null.equals(x) // Исключение - NullPointerException
>```

**Почему `equals()` при сравнении с `null`[^1] возвращает `false`?**
    - Согласно стандартному контракту `equals()`, объект может быть равен только объекту того же типа (или совместимого типа).
    - `null` не является экземпляром какого-либо класса, поэтому он не может быть равен никакому объекту.
