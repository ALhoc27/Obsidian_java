```java
import java.lang.reflect.Method;

class Example {
    public void publicMethod() {}
    private void privateMethod() {}
}

public class Main {
    public static void main(String[] args) {
        Method[] methods = Example.class.getMethods();

        for (Method method : methods) {
            System.out.println("Метод: " + method.getName());
        }
    }
}
```
**Результат**
```yaml
Метод: publicMethod
Метод: equals
Метод: toString
Метод: hashCode
Метод: getClass
Метод: notify
Метод: notifyAll
Метод: wait
Метод: wait
Метод: wait
```

#### **1. Метод `publicMethod`**

- Это единственный публичный метод, который **объявлен в классе `Example`**.
- Поскольку он публичный, он включается в результат.

#### **2. Методы из класса `Object`**

Класс `Example` неявно наследуется от класса `Object`. Все его публичные методы автоматически доступны для экземпляров `Example`. Эти методы включают:

- **`equals`**: Проверяет равенство объектов.
- **`toString`**: Возвращает строковое представление объекта.
- **`hashCode`**: Возвращает хэш-код объекта.
- **`getClass`**: Возвращает объект `Class`, представляющий класс объекта.
- **`notify`, `notifyAll`**: Используются для управления потоками (уведомление потоков, ожидающих на мониторе объекта).
- **`wait` (три версии)**: Заставляет поток ждать определенное время или до уведомления.

#### **3. Приватные методы (`privateMethod`)**

- Метод `privateMethod` не отображается, потому что:
    - `getMethods()` возвращает **только публичные методы**.
    - Приватные методы не наследуются и недоступны через этот метод.

---

### **Почему три версии `wait`?**

Метод `wait` в классе `Object` имеет **три перегруженные версии**:

1. `wait()` — ждет, пока другой поток не вызовет `notify` или `notifyAll`.
2. `wait(long timeout)` — ждет заданное количество миллисекунд.
3. `wait(long timeout, int nanos)` — ждет заданное количество миллисекунд и наносекунд.

Каждая версия считается отдельным методом, поэтому все три включены в результат.
