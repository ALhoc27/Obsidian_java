Класс `Service` в Android — это базовый класс, который предназначен для выполнения долгих фоновых операций в Android-приложении.
### Основные особенности `Service`

- **Фоновая работа без интерфейса**: `Service` не отображает UI и работает в фоновом режиме.
- **Жизненный цикл**: Сервис управляется системой Android через специальные методы жизненного цикла.
- **Взаимодействие с компонентами**: Сервис может быть запущен методом `startService()` и привязан к компонентам методом `bindService()`.
- **Foreground Service**: Это сервис, работающий в foreground-режиме, что предотвращает его завершение системой и позволяет уведомлять пользователя о его работе.

### Типы сервисов
Сервисы бывают двух основных типов:

1. **Started Service** (запущенный сервис):
    - Запускается методом `startService(Intent intent)` и выполняет задачу до тех пор, пока не завершится самостоятельно с вызовом `stopSelf()` или не будет остановлен вызовом `stopService()`.
    - Подходит для задач, которые должны выполняться некоторое время (например, загрузка файла).
2. **Bound Service** (сервис привязки):
    - Создается для взаимодействия с другими компонентами, такими как Activity, с помощью метода `bindService(Intent intent, ServiceConnection conn, int flags)`.
    - Работает только до тех пор, пока к нему привязаны другие компоненты. Когда компонент отсоединяется, сервис останавливается.


> [!TIP] Жизненный цикл `Service`
> В зависимости от того, как сервис запускается — как `Started Service` или `Bound Service` — его жизненный цикл различается.
> 
> Для **Started Service** важны методы: `onCreate()`, `onStartCommand()`, `onDestroy()`. Для **Bound Service**: `onCreate()`, `onBind()`, `onUnbind()`, `onRebind()` и `onDestroy()`.

### Методы жизненного цикла `Service`

- **`onCreate()`**[^1]: Вызывается при создании сервиса. Здесь обычно выполняется начальная настройка, например, инициализация переменных или настройка уведомлений.
- [[onStartCommand(Intent intent, int flags, int startId)]][^2]: Вызывается каждый раз, когда компонент запускает сервис методом `startService()`. Метод обрабатывает запрос и определяет, должен ли сервис продолжать работу или быть перезапущен в случае завершения.
- [[onBind(Intent intent)]]: Используется для привязки сервиса к компоненту. Если сервис не предназначен для привязки, метод возвращает `null`.
- [[onUnbind(Intent intent)]]: Вызывается, когда все клиенты отсоединяются от привязанного сервиса.
- [[onRebind(Intent intent)]]: Вызывается, если сервис уже был отсоединен, но теперь к нему снова подключается клиент.
- [[onDestroy()]]: Вызывается перед уничтожением сервиса, когда он завершает свою работу. Здесь освобождаются ресурсы.

##### ⚙︎ [[Пример _ Запущенный сервис с повторяющейся задачей]] #####
### Пример использования `Service`
Вот простой пример сервиса, который будет записывать сообщение в лог каждые 5 секунд:

```java
public class MyService extends Service {
    
    private Handler handler;
    private Runnable runnable;

    @Override
    public void onCreate() {
        super.onCreate();
        handler = new Handler();
        runnable = new Runnable() {
            @Override
            public void run() {
                Log.d("MyService", "Service is running...");
                handler.postDelayed(this, 5000); // Повторять каждые 5 секунд
            }
        };
        handler.post(runnable); // Запуск задачи
    }

    @Override
    public int onStartCommand(Intent intent, int flags, int startId) {
        Log.d("MyService", "Service started");
        return START_STICKY; // Сервис будет перезапущен, если система его завершит
    }

    @Override
    public IBinder onBind(Intent intent) {
        return null; // Сервис не привязывается к Activity
    }

    @Override
    public void onDestroy() {
        super.onDestroy();
        handler.removeCallbacks(runnable); // Остановка задачи
        Log.d("MyService", "Service stopped");
    }
}
```
### Применение `START_STICKY`, `START_NOT_STICKY` и `START_REDELIVER_INTENT`

- **`START_STICKY`**: Сервис будет перезапущен после его завершения системой, но `intent` не передастся.
- **`START_NOT_STICKY`**: Сервис не перезапускается системой, если его завершили из-за нехватки ресурсов.
- **`START_REDELIVER_INTENT`**: Сервис перезапустится системой и заново получит последний `intent`.

Класс `Service` является основой для создания сложных фонов задач, которые выполняются долго, и позволяет контролировать использование ресурсов устройства.



[^1]:#### `onCreate()`
	- Вызывается единожды при создании сервиса. Этот метод выполняется перед `onStartCommand()` и `onBind()` и используется для начальной настройки.
	- В этом методе инициализируются переменные, запускаются ресурсы и выполняется другая подготовительная работа.
	```java
	@Override
	public void onCreate() {
	    super.onCreate();
	    // Инициализация ресурсов или задач
	}
	```

[^2]:#### `onStartCommand(Intent intent, int flags, int startId)`
	- Вызывается каждый раз, когда компонент вызывает `startService()`.
	- Содержит логику, которая выполняется в сервисе, например, обработку данных или запуск фоновой задачи.
	- Возвращает один из трёх режимов перезапуска сервиса:
	    - **`START_STICKY`**: Сервис перезапускается автоматически после его завершения, но `intent` будет `null`. Подходит для долгосрочных сервисов.
	    - **`START_NOT_STICKY`**: Сервис не будет перезапущен автоматически. Подходит для короткоживущих задач.
	    - **`START_REDELIVER_INTENT`**: Сервис перезапустится и снова получит последний `intent`. Используется, если сервис критичен и намерен завершить задачу даже после перезапуска.
	```java
	@Override
	public int onStartCommand(Intent intent, int flags, int startId) {
	    // Выполнение задачи
	    return START_STICKY; // Указывает, что сервис должен быть перезапущен
	}
	```
