#### **Что такое дедупликация строк?**
Дедупликация строк — это процесс поиска и удаления дубликатов строк, которые существуют в Heap (но не в [[String Pool]]). JVM объединяет одинаковые строки, чтобы минимизировать потребление памяти.

Это происходит автоматически, если включён **GC (Garbage Collector) String Deduplication**, который ищет дубли строк и перенаправляет ссылки на единственный экземпляр.
#### **Включение дедупликации строк**
String Deduplication доступна в **Garbage Collector G1GC**, начиная с Java 8 (update 20 и выше). Чтобы включить её, нужно передать следующие аргументы JVM:
```java
-XX:+UseG1GC -XX:+UseStringDeduplication
```
#### Опциональные аргументы:
- `-XX:StringDeduplicationAgeThreshold=N`: задаёт минимальный "возраст" объекта (в сборках GC), после которого строка будет считаться кандидатом для дедупликации (по умолчанию — 3 сборки).

#### **Как проверить работу дедупликации?**
1. **Добавьте аргументы JVM:**
```java
java -XX:+UseG1GC -XX:+UseStringDeduplication -Xlog:gc* MyApplication
```
2. **Посмотрите лог GC:**  
Если дедупликация активирована, вы увидите сообщения вроде:
```java
[GC remark, 0.003 secs] String deduplication: 1000 candidates, 500 duplicates, 200 bytes saved
```

***Пример работы со строками в пуле:***
```java
public class StringDeduplicationExample {
    public static void main(String[] args) {
        String s1 = "Hello";       // добавляется в String Pool
        String s2 = "Hello";       // указывает на тот же объект в String Pool
        String s3 = new String("Hello"); // создаёт новый объект в Heap

        // Проверка ссылок
        System.out.println(s1 == s2); // true (один объект в String Pool)
        System.out.println(s1 == s3); // false (разные объекты)

        // Принудительное добавление в пул
        String s4 = s3.intern();      
        System.out.println(s1 == s4); // true (оба теперь в String Pool)
    }
}
```
#### **Когда дедупликация полезна?**
- Если ваше приложение активно работает со строками (например, парсинг, обработка JSON/XML).
- Когда строки в Heap имеют много дубликатов, дедупликация может значительно снизить использование памяти.
#### **Ограничения:**
- Дедупликация потребляет ресурсы процессора, поэтому в приложениях с низкими требованиями к памяти она может быть избыточной.
- Работает только с G1GC.
