### **Class Loaders в Java**

**Class Loader** — это компонент JVM, который отвечает за загрузку классов в память во время выполнения программы. Классы загружаются из файлов `.class`, JAR-архивов, сетевых ресурсов или других источников. Этот механизм обеспечивает гибкость и безопасность при работе с динамическими приложениями.

### **Зачем нужны Class Loaders**

1. **Динамическая загрузка классов:**  
    Классы загружаются по мере необходимости, а не все сразу при запуске программы.
2. **Поддержка модульности:**  
    Позволяет изолировать загрузку классов разных модулей, например, плагинов.
3. **Гибкость работы с ресурсами:**  
    Классы могут загружаться из разных источников (локальные файлы, удалённые сервера).
4. **Безопасность:**  
    Каждый `Class Loader` имеет свою область видимости и обеспечивает защиту классов от нежелательных изменений.

### **Виды Class Loaders**
![[Снимок экрана 2024-12-10 в 22.43.43 1.png|650]]

1. **Bootstrap Class Loader**
    - Это первичный загрузчик, встроенный в JVM.
    - Загрузает основные классы языка Java из библиотеки JDK (`rt.jar` или модулей в JDK 9+).
    - Работает на уровне нативного кода.
    - Не является объектом Java (нет соответствующего класса в Java).

2. **Extension Class Loader (Platform Class Loader)**
    - Загружает классы из стандартных расширений JDK, расположенных в папке `lib/ext` или указанных через `java.ext.dirs`.
    - Начиная с Java 9, заменён Platform Class Loader, который выполняет схожую функцию.

3. **Application Class Loader (System Class Loader)**
    - Загружает пользовательские классы и библиотеки, указанные в `CLASSPATH`.
    - Это основной загрузчик для большинства приложений.

4. * ***Custom Class Loaders (Пользовательские загрузчики)**
    - Вы можете создать свой загрузчик классов, унаследовав его от `ClassLoader`.
    - Используется для специфических целей, например:
        - Загрузка классов из нестандартных мест (сети, базы данных).
        - Реализация плагинов.
        - Контроль версий библиотек.

### **Принципы работы Class Loaders** ==(кратко)==

1. **Иерархия загрузчиков (Delegation Model):**  
    Загрузка класса начинается с **родительского загрузчика**:
    - Сначала `Bootstrap Class Loader`.
    - Если родительский загрузчик не может найти класс, он делегирует задачу своему потомку.
    - Если ни один загрузчик не может найти класс, выбрасывается `ClassNotFoundException`.
      
2. **Область видимости:**  
    Каждый загрузчик имеет свою область видимости. Классы, загруженные разными загрузчиками, могут быть изолированы друг от друга.

### **Основные выводы**
- **Class Loader** отвечает за загрузку классов в память.
- Реализует **иерархическую модель делегирования**.
- Позволяет загружать классы из разных источников, обеспечивая безопасность и изоляцию.
- Создание собственных загрузчиков открывает широкие возможности для динамической работы с классами и ресурсами.

### **Принципы работы Class Loaders** ==(подробнее)==  4

> Class Loaders работают по определённым правилам, обеспечивающим безопасность, изоляцию классов и эффективное управление памятью. Главным принципом является **модель делегирования** (Delegation Model), которая определяет, как классы ищутся и загружаются.
> 
>  ==**1. Модель делегирования (Delegation Model)**==
> 
>  **Описание**
> Каждый Class Loader не загружает класс самостоятельно, если этот класс уже может быть загружен его родительским загрузчиком. Это обеспечивает:
> 
> - **Изоляцию:** Нижестоящие загрузчики не могут перезаписать классы, загруженные вышестоящими.
> - **Целостность:** Стандартные и базовые классы (например, `java.lang.String`) всегда загружаются одним и тем же Class Loader.
> 
> **Алгоритм делегирования**
> 	1. Class Loader получает запрос на загрузку класса (например, `java.lang.String`).
> 	2. Он передаёт запрос своему **родительскому Class Loader**.
> 	3. Если родительский загрузчик находит класс, он возвращает его.
> 	4. Если родительский загрузчик не может найти класс, текущий Class Loader пытается загрузить его самостоятельно.
> 	5. Если ни один загрузчик не находит класс, выбрасывается `ClassNotFoundException`.
> 
> ==**2. Иерархия Class Loaders**==
> Class Loaders организованы в виде дерева (иерархии). Вершина иерархии — это Bootstrap Class Loader.
> 
> **Основные уровни:**
> 1. **Bootstrap Class Loader:**
> 	- Загружает базовые классы JVM из JDK, такие как java.lang.String и java.util.HashMap.
> 	- Работает на уровне нативного кода.
> 	- Не имеет родительского загрузчика.
>       
> 2. **Platform (или Extension) Class Loader:**
> 	- Загружает расширения и библиотеки Java из папки lib/ext или других указанных мест.
> 	- Его родителем является Bootstrap Class Loader.
> 
> 3. **Application (или System) Class Loader:**
> 	- Загружает пользовательские классы и библиотеки, указанные в CLASSPATH.
> 	- Его родителем является Platform Class Loader.
> 
> 4. **Custom Class Loader:**
> 	- Пользовательские загрузчики, которые могут наследоваться от ClassLoader или URLClassLoader.
> 	- Обычно используются для загрузки классов из нестандартных мест (сетевых ресурсов, баз данных и т.д.).
> 
> ==**3. Область видимости классов**==
> Каждый Class Loader имеет собственную область видимости:
> - Классы, загруженные определённым Class Loader, видны только этому загрузчику и его потомкам.
> - Классы, загруженные предком, доступны потомкам, но не наоборот.
> 
> **Пример:**
> Если два разных Custom Class Loaders загружают один и тот же класс из разных источников, эти классы считаются **разными**, даже если их байт-код идентичен.
> 
> ==**4. Процесс загрузки класса**==
> Загрузка класса состоит из трёх этапов:
> 1. **Loading (Загрузка):**
> 	1. Class Loader ищет `.class` файл в указанном месте (например, CLASSPATH). c помощью модели делигирования
> 	2. Загружает байт-код в память.
> 	3. Если класс не найден, выбрасывается `ClassNotFoundException`.
> 
> 2. **Linking (Связывание):**
> 	1. **Verification (Проверка):** Проверка байт-кода на соответствие спецификации JVM.
> 	2. **Preparation (Подготовка):** Выделение памяти для статических полей и инициализация их значениями по умолчанию.
> 	3. **Resolution (Разрешение):** Привязка ссылок на другие классы.\
> 	
> 3. **Initialization (Инициализация):**
> 	1. Выполняются статические блоки инициализации (`static {}`) и инициализация статических полей.
> 
> 
> ==**5. Преимущества модели делегирования**==
> 1. **Безопасность:**  
> 	- Базовые классы, такие как `java.lang.String`, всегда загружаются одним и тем же загрузчиком, что предотвращает атаки через подмену классов.
> 
> 2. **Изоляция:**  
> 	- Разные модули или плагины могут использовать свои версии библиотек, не конфликтуя друг с другом.
> 
> 3. **Повторное использование:**  
> 	- Классы, загруженные предками, могут быть переиспользованы, что экономит память и ускоряет выполнение.


### **Ограничения и ошибки**

1. **ClassNotFoundException:**  
    Если класс не найден в доступных Class Loaders.
    
2. **NoClassDefFoundError:**  
    Если класс найден, но его загрузка завершилась с ошибкой (например, из-за неверного байт-кода).
    
3. **LinkageError:**  
    Если возникает конфликт между двумя версиями одного класса.
