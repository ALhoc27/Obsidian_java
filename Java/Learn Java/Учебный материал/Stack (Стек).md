Стек в Java представляет собой область памяти, используемую для хранения вызовов методов, локальных переменных и временных данных. Каждая нить выполнения (поток) получает свой отдельный стек, что позволяет изолировать выполнение методов между потоками.

| Что такое **стек**? Стековая память в Java работает по схеме **LIFO** (Последний-зашел-Первый-вышел). Всякий раз, когда вызывается метод, в памяти стека создается новый блок, который содержит примитивы и ссылки на другие объекты в методе. Как только метод заканчивает работу, блок также перестает использоваться, тем самым предоставляя доступ для следующего метода.<br><br>==Размер стека намного меньше объема памяти в куче.==<br><br>Всякий раз, когда создается объект, он всегда хранится в куче, а в памяти стека содержится ссылка на него. Память стека содержит только локальные переменные примитивных типов и ссылки на объекты в куче.<br><br>Стек обеспечивает быстрый доступ к данным и автоматическое управление памятью, но его размер ограничен. Если программа потребляет больше стековой памяти, чем доступно, это может привести к ошибке переполнения стека.<br><br>\| ![[PROgrammer-Stack_1.gif \| 350]] \| ![[Pasted image 20241215220307.png \| 350]] \|<br>\| ------------------------- \| ----- \| ---------------------------------- \| ----- \|<br><br>Когда вызывается функция, для неё выделяется блок памяти на вершине стека. Этот блок, известный как “**фрейм стека**”, содержит пространство для всех локальных переменных функции, а также информацию, такую как адрес возврата — место в коде, куда программа должна вернуться после завершения функции.<br><br>Когда одна функция вызывает другую, для новой функции выделяется собственный фрейм стека, и она становится текущей активной функцией. По завершении работы функции её фрейм стека удаляется, и управление передаётся обратно вызывающей функции.<br><br>![[0bc79f5d-da3d-4060-a2a2-a7f2060cc4a6.mp4]]<br>*Так выглядят фреймы стека в режиме дебага в Idea*<br><br><br><span style="font-size: 1.2em; font-weight: bold; margin: 0; line-height: 1.2; display: inline-block; margin-bottom: 10px;"> &nbsp;&nbsp;Особенности работы</span><br>- **LIFO (Last-In, First-Out)**: Стек работает по принципу “последним пришёл — первым ушёл”. Это означает, что последняя вызванная функция будет первой, которая завершит работу и вернёт управление.<br>- **Автоматическое управление памятью**: Когда функция завершает выполнение, её локальные переменные автоматически удаляются. Это упрощает управление памятью, так как нет необходимости в явном освобождении памяти.<br>- **Ограниченный размер**: Размер стека обычно ограничен. Если программа попытается использовать больше памяти, чем доступно в стеке, это приведёт к ошибке переполнения стека.<br>- **Быстрый доступ**: Доступ к данным в стеке обычно быстрее, чем к данным в куче, поскольку стек локализован в памяти, и данные из него могут загружаться в кэш процессора для ускоренного доступа.<br><br> |

### **Состав стека**
1. **Фреймы стека (Stack Frames)**  
    Стек состоит из фреймов, каждый из которых создаётся при вызове метода и уничтожается после завершения метода.  
    **Фрейм стека включает:**
    - **Локальные переменные:**
        - Переменные, объявленные внутри метода, включая параметры метода.
        - Примитивные типы данных (`int`, `double`, `char` и т. д.) и ссылки на объекты.
    - **Операнды (Operand Stack):**
        - Используется для временного хранения данных во время вычислений.
        - Например, промежуточные результаты математических операций.
    - **Метаданные метода:**
        - Адрес возврата (где продолжить выполнение после завершения метода).
        - Информация о методе (например, его идентификатор).
2. **Локальные переменные** Хранят значения, которые используются только внутри метода.
	*Пример:*
```java
void method(int x) {
    int y = 5; // Локальная переменная
    int z = x + y; // z также локальная переменная
}
```
`x`, `y`, `z` находятся в стеке, потому что они принадлежат методу.
3. **Аргументы метода** При вызове метода его параметры копируются в стек.  
	*Пример:*
```java
void add(int a, int b) {
    int sum = a + b; // a, b и sum находятся в стеке
}
```
4. **Адрес возврата**
	- Указывает место в коде, куда нужно вернуться после завершения метода.
	- Если метод вызывает другой метод, текущий адрес сохраняется в фрейме стека.
1. **Операнды**
    - Временные данные, участвующие в вычислениях.
    - Например, для операции `a + b` операнды `a` и `b` помещаются в стек, а затем результат вычисления.
