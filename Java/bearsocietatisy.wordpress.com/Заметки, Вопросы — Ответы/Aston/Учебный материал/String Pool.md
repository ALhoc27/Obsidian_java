**String Pool** — это специальная область памяти в Java, где хранятся строки. Основная цель String Pool — экономия памяти и оптимизация работы с неизменяемыми строками (объектами `String`).

### Как это работает?
1. **Когда вы создаёте строку с помощью литерала:**
```java
String str1 = "Hello"; 
String str2 = "Hello";
```

- Одна строка `"Hello"` создаётся в String Pool.
- Переменные `str1` и `str2` будут ссылаться на один и тот же объект.


2. **Когда строка создаётся через `new`:**
```java
String str3 = new String("Hello");
```
Создаётся новый объект строки в куче, даже если `"Hello"` уже есть в String Pool.

### Почему String Pool важен?
1. **Экономия памяти.**
    - Повторяющиеся строки не создаются заново, если они уже есть в String Pool.
```java
String a = "Java";
String b = "Java";
System.out.println(a == b); // true (оба указывают на один объект)
```

2. **Быстрая проверка строк.**
- Сравнение строк с помощью оператора `==` работает быстрее, так как сравниваются ссылки, а не содержимое.

### Где хранится String Pool?
- До **Java 7**: String Pool находился в **Permanent Generation** (часть кучи).
- С **Java 7 и новее**: String Pool перенесён в **Metaspace** (нативная память).


### Важные методы
1. **Метод `intern()`:**
	- Перемещает строку в String Pool или возвращает существующую строку из пула.
*Пример:*
```java
String s1 = new String("Hello");
String s2 = "Hello";
System.out.println(s1 == s2);         // false (разные объекты)
System.out.println(s1.intern() == s2); // true (ссылка на один объект из пула)
```

### Основные особенности String Pool
1. **Строки неизменяемы.**
    - Как только строка создаётся, её содержимое нельзя изменить.
    - Это делает использование String Pool безопасным.

2. **Используется только для строк, созданных как литералы.**
    - Объекты, созданные через `new`, не попадают в пул автоматически.

3. **Оптимизация ресурсоёмких операций.**
    - String Pool используется для экономии памяти при работе с большим количеством одинаковых строк.
