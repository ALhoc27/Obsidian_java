
### Абстрактные классы и интерфейсы: сравнение и особенности
Абстрактные классы и интерфейсы — это механизмы, которые используются в объектно-ориентированном программировании (ООП) для проектирования архитектуры приложений, обеспечивая гибкость и повторное использование кода. Хотя они схожи в своей цели, между ними есть ключевые различия.

В языке программирования Java **абстрактные классы** и **интерфейсы** предоставляют механизмы для определения общего поведения классов и поддерживают концепцию полиморфизма и наследования. Рассмотрим их подробно.

### **Абстрактные классы**
**Абстрактные классы в Java** – это ключевые элементы объектно-ориентированного программирования, которые позволяют создавать иерархии классов и обеспечивают возможность инкапсуляции общих свойств и методов.
#### Описание:
Абстрактный класс — это класс, который не может быть инстанцирован напрямую. Он может содержать как абстрактные методы (без реализации), так и методы с реализацией. ==Используется для описания общих свойств и поведения группы классов.==

#### Что такое абстрактный класс?
Абстрактный класс – это класс, который содержит хотя бы один **абстрактный метод**. Абстрактный метод – это метод без реализации, то есть у него нет тела метода (`{}`). Такие методы объявляются с ключевым словом `abstract`.

Абстрактный класс — это класс, который не может быть непосредственно инстанцирован (создан в виде объекта). Он служит базой для других классов и может содержать как абстрактные методы (без реализации), так и методы с реализацией.

```java
public abstract class Animal {
    public abstract void makeSound();
}
```
 
> В приведенном примере `Animal` является абстрактным классом, а метод `makeSound()` – абстрактным методом. Это означает, что любой класс, наследующийся от `Animal`, обязан реализовать этот метод, иначе он также станет абстрактным.

#### Зачем нужны абстрактные классы?

Абстрактные классы используются для того, чтобы:
1. ==**Инкапсулировать общие свойства и поведение** между несколькими классами==.Например, у всех животных могут быть такие общие характеристики, как имя, возраст и вес. Эти данные можно вынести в базовый абстрактный класс.
2. **Обеспечить обязательную реализацию определенных методов** в подклассах.Например, каждый вид животного может издавать свой звук, но способ его воспроизведения различается. Поэтому метод `makeSound()` остается абстрактным, а реализация этого метода будет определяться в конкретных классах-наследниках.
3. **Создать шаблон поведения**, оставляя конкретную реализацию некоторым частям кода на усмотрение разработчиков.
4. **Упростить кодовую базу**, избегая дублирования одинаковых участков кода.

*Пример абстрактного класса:*
```java
public abstract class Shape {
    private String color;
    
    // Конструктор
    public Shape(String color) {
        this.color = color;
    }
    
    // Обычный метод
    public String getColor() {
        return color;
    }
    
    // Абстрактный метод
    public abstract double calculateArea();
}
```
Здесь класс `Shape` имеет обычный метод `getColor()` и абстрактный метод `calculateArea()`, который должны будут реализовать конкретные классы-наследники.

#### Как использовать абстрактные классы?
Чтобы использовать абстрактный класс, необходимо создать конкретный класс, который наследуется от абстрактного и реализует все его абстрактные методы.

*Пример:*
```java
public class Circle extends Shape {
    private double radius;
    
    public Circle(double radius, String color) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
}
```
Класс `Circle` наследует от `Shape` и реализует метод `calculateArea()`, специфичный для круга.

#### Особенности абстрактных классов
1. **Абстрактный класс нельзя инстанцировать**.То есть невозможно создать экземпляр абстрактного класса непосредственно:
```java
Shape shape = new Shape("blue"); // Ошибка компиляции!
```
2. **Подклассы обязаны переопределять все абстрактные методы** родительского класса, если они сами не являются абстрактными.Если какой-то из абстрактных методов не реализован в подклассе, то этот подкласс тоже становится абстрактным.
3. **Абстрактные классы могут содержать конструкторы**.Они необходимы для инициализации полей класса и вызова конструктора суперкласса при создании экземпляра конкретного подкласса.
4. **Абстрактные классы могут иметь поля и обычные методы**.Это позволяет инкапсулировать общие данные и логику, которая должна быть доступна всем подклассам.

#### **Особенности v2**:
1. **Абстрактные методы**: Методы без реализации, которые должны быть реализованы в производных классах.
2. **Обычные методы**: Методы с реализацией, которые могут быть переопределены.
3. **Частичные реализации**: Абстрактный класс может содержать частичную реализацию, которую наследники могут использовать или переопределять.
4. **Модификаторы доступа**: Поддерживаются (public, protected, private).
5. **Поля и свойства**: Может содержать поля и свойства.
6. **Конструкторы**: Можно определить конструкторы, которые будут вызываться при создании объекта производного класса.

#### Полиморфизм и абстрактные классы
Абстрактные классы часто используются вместе с полиморфизмом. Полиморфизм позволяет ссылаться на объекты различных типов через одну и ту же ссылку. Это особенно полезно, когда нужно работать с объектами разных классов, имеющих общий интерфейс или абстрактный класс.

*Пример:*
```java
Shape[] shapes = new Shape[2];
shapes[0] = new Circle(10, "red");
shapes[1] = new Rectangle(5, 7, "green");

for (Shape shape : shapes) {
    System.out.println(shape.getColor());
    System.out.println(shape.calculateArea());
}
```
Здесь массив `shapes` хранит ссылки на объекты типа `Shape`, но фактически эти объекты могут быть экземплярами различных классов-наследников (`Circle`, `Rectangle` и др.). При вызове метода `calculateArea()` будет выполняться соответствующая реализация каждого конкретного класса.

#### Как абстрактные классы связаны с интерфейсом?
Абстрактные классы и интерфейсы в Java тесно взаимосвязаны, поскольку оба этих механизма ==служат для определения общего поведения объектов,== однако они имеют свои различия и применяются в зависимости от контекста задачи.

#### Основные отличия абстрактных классов и интерфейсов

##### 1. Реализация методов
- **Абстрактные классы**: Могут содержать как абстрактные методы (без реализации), так и обычные методы с реализацией. Подкласс обязан реализовать только те методы, которые определены как абстрактные.
- **Интерфейсы**: До версии Java 8 все методы в интерфейсе были абстрактными (то есть без реализации). Начиная с Java 8, интерфейсы могут содержать методы с реализацией (методы по умолчанию и статические методы).

##### 2. Наследование
- **Абстрактные классы**: Класс может наследоваться только от одного абстрактного класса (как и от любого другого класса). Это связано с тем, что в Java множественное наследование классов запрещено.
- **Интерфейсы**: Класс может реализовывать любое количество интерфейсов одновременно. Таким образом, интерфейсы дают больше свободы в комбинировании функциональностей.

##### 3. Поля
- **Абстрактные классы**: Могут содержать переменные-члены (поля) с любыми уровнями доступа (private, protected, public). Поля могут быть изменяемыми или неизменяемыми.
- **Интерфейсы**: Все поля в интерфейсе автоматически считаются `public static final`, то есть они всегда являются константами. До Java 9 в интерфейсах вообще не было возможности определять переменные-члены.

##### 4. Модификаторы методов
- **Абстрактные классы**: Методы могут иметь любые уровни доступа (private, protected, public).
- **Интерфейсы**: Все методы до Java 9 считались `public abstract` (если не были методами по умолчанию или статическими). С версии Java 9 появилась возможность указывать уровень доступа к методам интерфейса.

##### 5. Конструкторы
- **Абстрактные классы**: Могут иметь конструкторы, которые используются для инициализации полей класса.
- **Интерфейсы**: Не содержат конструкторов.

#### Связь между абстрактными классами и интерфейсами

Абстрактные классы и интерфейсы часто работают совместно, обеспечивая различные аспекты проектирования системы:

1. **Определение базовой структуры и общих методов**:
    - Абстрактные классы обычно используются для предоставления общей структуры и базовых методов, которые могут быть полезны сразу нескольким классам.
    - Интерфейсы определяют контракт поведения, которому должны следовать классы, реализующие данный интерфейс.
2. **Комбинирование функционала**:
    - Классы могут расширять абстрактные классы и одновременно реализовывать несколько интерфейсов, что дает большую гибкость в проектировании.
3. **Полиморфизм**:
    - Оба подхода поддерживают полиморфизм, позволяя ссылкам на абстрактные классы или интерфейсы указывать на объекты различных конкретных классов.
#### Пример использования абстрактного класса и интерфейса
Рассмотрим пример, где абстрактный класс определяет общие черты фигуры, а интерфейс добавляет дополнительную функциональность:
```java
// Абстрактный класс
public abstract class Shape {
    private String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    public String getColor() {
        return color;
    }
    
    public abstract double calculateArea();
}

// Интерфейс
public interface Movable {
    void move(int x, int y);
}

// Конкретная фигура
public class Circle extends Shape implements Movable {
    private double radius;
    
    public Circle(double radius, String color) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public void move(int x, int y) {
        System.out.println("Перемещаем круг на (" + x + ", " + y + ")");
    }
}
```

### Интерфейсы
Интерфейс в Java — это ключевой элемент языка, ==**используемый для описания контракта поведения**==, которому должны соответствовать классы, реализующие этот интерфейс. Он предоставляет набор методов, которые должны быть реализованы в классе, и таким образом служит основой для обеспечения совместимости и взаимодействия между различными компонентами программы.

### **Что это?**
Интерфейс — это коллекция абстрактных методов (и других членов с версии Java 8+), которая определяет, что класс _должен_ делать, но не _как_ он это делает. Интерфейсы задают контракт, который обязаны соблюдать их реализации.

### Определение интерфейса

Интерфейс определяется с помощью ключевого слова `interface`. В отличие от классов, интерфейсы не могут содержать реализацию методов (до Java 8), а лишь их сигнатуры. Также интерфейсы не могут иметь конструкторов и состояния (переменных-членов).

*Пример простого интерфейса:*
```java
public interface Movable {
    void move(int x, int y);
}
```
Этот интерфейс требует, чтобы любой класс, реализующий его, имел метод `move` с двумя целочисленными параметрами.

### Ключевые особенности интерфейсов
1. **Методы интерфейса**:
    - До Java 8 все методы интерфейса были абстрактными. Даже если не указывать модификаторы `public` и `abstract`, компилятор сам подразумевал их наличие. После ==введения методов  **в Java 8 по умолчанию `default` и статических методов `static`, которые могут иметь тело**,== если метод не помечен как `default` или `static`, он считается абстрактным. 
    -  начиная с Java 9, в интерфейсах можно задавать модификаторы доступа (`private`, `protected`) для методов по умолчанию и статических методов.
      ==(**Методы интерфейса**: (**`default`, `static`**) с модификатором `public` + с Java 9 (`private`, `protected`) , и **`abstract`** с модификатором `public`)==
1. **Поля интерфейса**:
    - Все поля интерфейса автоматически считаются `public`, `static` и `final`, то есть они ==являются константами==.
    - Поля интерфейса обычно используются для хранения глобальных значений, таких как константы.
2. **Реализация интерфейса**:
    - Для того чтобы класс мог использовать интерфейс, он должен реал
    - изовать его с помощью ключевого слова `implements`.
    - Класс, реализующий интерфейс, обязан предоставить реализацию всех абстрактных методов этого интерфейса.
3. **Наследование интерфейсов**:
    - Один интерфейс может наследовать другой интерфейс с использованием ключевого слова `extends`.
    - Класс может реализовывать несколько интерфейсов одновременно.
4. **Модификаторы доступа**:
    - До Java 9 все методы интерфейса были `public`. Однако начиная с Java 9, в интерфейсах можно задавать модификаторы доступа (`private`, `protected`) для методов по умолчанию и статических методов.
5. **Методы по умолчанию (default methods)**:
    - Введены в Java 8. Позволяют добавлять методы с реализацией прямо в интерфейс, не нарушая совместимость с уже существующими классами.
    - Методы по умолчанию помечаются ключевым словом `default`.
6. **Статические методы**:
    - Также введены в Java 8. Статические методы могут быть определены внутри интерфейса и доступны через имя интерфейса.

### Еще раз  **Особенности**

1. **Объявление**: Интерфейс объявляется с использованием ключевого слова `interface`.
```java
public interface Flyable {
    void fly(); // Метод без реализации
}
```
    
2. **Методы**:
    - До Java 8: методы в интерфейсах всегда были абстрактными и публичными.
    - С Java 8:
        - Можно добавлять **default-методы** (с реализацией, используя ключевое слово `default`).
        - Можно добавлять **static-методы**.
    - С Java 9: можно добавлять **private-методы** (используются для помощи другим методам интерфейса).
```java
public interface Vehicle {
    default void start() {
        System.out.println("Starting the vehicle...");
    }

    static void showDetails() {
        System.out.println("This is a vehicle interface.");
    }
}
```
    
3. **Поля**:
    - Поля интерфейсов всегда являются **`public static final`**, даже если это явно не указано.
```java
int MAX_SPEED = 120; // Эквивалентно public static final int MAX_SPEED = 120;
```
    
4. **Наследование**:
    - Класс может реализовывать несколько интерфейсов (в отличие от наследования, где возможен только один родительский класс).
    - Интерфейсы могут наследовать друг друга.
```java
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("Bird is flying");
    }
}
```
    
5. **Использование**: Интерфейсы применяются для задания контрактов или API, которые должны реализовывать различные классы.

### **Выбор:** абстрактный класс или интерфейс
1. Используйте **абстрактный класс**, если:
    - У вас есть общая логика, которую должны унаследовать подклассы.
    - Классы тесно связаны по смыслу.
    - Вы хотите предоставить общие поля или методы.

1. Используйте **интерфейс**, если:
    - Вам нужно определить контракт, который может быть реализован разными, несвязанными классами.
    - Требуется множественное наследование.
    - Вы хотите создать гибкую архитектуру для реализации различных поведений.

Комбинация обоих подходов также часто используется в сложных проектах.
