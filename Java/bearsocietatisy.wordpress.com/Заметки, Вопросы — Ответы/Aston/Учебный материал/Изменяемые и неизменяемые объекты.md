В Java объекты можно разделить на **изменяемые (mutable)** и **неизменяемые (immutable)**. Понимание разницы между ними важно для эффективного проектирования кода, управления памятью и обеспечения безопасности многопоточного доступа.

## **Изменяемые объекты (Mutable Objects)**
Изменяемые объекты — это объекты, состояние которых можно изменить после их создания.
#### **Примеры:**
1. **Коллекции (базовые реализации):**
    - `ArrayList`
    - `LinkedList`
    - `HashMap`
    - `HashSet`
    - `TreeMap`
    - `TreeSet`

1. **Массивы (`Array`):**    
    - Вы можете изменять элементы массива по индексу.
```java
int[] arr = {1, 2, 3};
arr[0] = 10; // Изменение элемента массива\
```
    
3. **Классы для работы со строками:**
    - `StringBuilder`
    - `StringBuffer`
```java
StringBuilder sb = new StringBuilder("Hello");
sb.append(" World"); // Изменение строки
```
    
4. **Дата и время (старые классы):**  [[Работа с датой в java]]
    - `Date`
    - `Calendar`
```java
Calendar cal = Calendar.getInstance();
cal.set(Calendar.YEAR, 2024); // Изменение года
```
    
5. **Кастомные классы:**
    - Если вы создаёте класс с методами для изменения состояния (например, сеттерами), он будет изменяемым.

#### **Особенности изменяемых объектов:**
- Предоставляют методы для изменения внутреннего состояния (например, сеттеры, добавление элементов, изменение значений).
- Используются, когда требуется часто изменять данные.
- Могут быть проблемными при работе в многопоточной среде, так как их состояние может быть изменено несколькими потоками одновременно.

Если класс содержит изменяемые объекты (например, коллекции или массивы), важно вернуть их копии или сделать эти объекты неизменяемыми.
***Пример с коллекцией:***
```java
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

final class ImmutableStudent {
    private final String name;
    private final List<String> subjects;

    public ImmutableStudent(String name, List<String> subjects) {
        this.name = name;
        // Создаем копию списка, чтобы избежать изменений извне
        this.subjects = new ArrayList<>(subjects);
    }

    public String getName() {
        return name;
    }

    // Возвращаем неизменяемую копию списка
    public List<String> getSubjects() {
        return Collections.unmodifiableList(subjects);
    }

    @Override
    public String toString() {
        return "ImmutableStudent{name='" + name + "', subjects=" + subjects + "}";
    }
}
```
Объяснение[^2]
### **Преимущества неизменяемых объектов**
1. **Потокобезопасность**: Неизменяемые объекты безопасны для работы в многопоточной среде без дополнительной синхронизации.
2. **Упрощённое тестирование**: Неизменяемые объекты легче тестировать, так как их состояние не изменяется.
3. **Кэширование**: Неизменяемые объекты могут быть кэшированы, так как их значения не меняются.
4. **Простота использования**: Уменьшается вероятность багов из-за непредвиденных изменений состояния.

### **Недостатки**
1. **Высокая стоимость создания**: При изменении состояния создаётся новый объект, что может привести к дополнительным затратам на память и производительность.
2. **Сложность при большом количестве изменяемых данных**: Неизменяемые объекты неудобны для работы с большими наборами данных, которые часто меняются.

*В таких случаях можно использовать изменяемые объекты или комбинировать неизменяемость с изменяемыми структурами.*

## **Неизменяемые объекты (Immutable Objects)**
Неизменяемые объекты не позволяют изменять своё состояние после создания.

**Неизменяемые объекты — это объекты, состояние которых нельзя изменить после их создания.** (не позволяют изменять своё состояние после создания). Любое изменение приводит к созданию нового объекта.

#### **Примеры:**
1. **Классы для строк:**
    - `String`
```java
String s = "Hello";
String newS = s.concat(" World"); // Создан новый объект
```
    
2. **Классы-обёртки для примитивов:**
    - `Integer`
    - `Double`
    - `Float`
    - `Boolean`
```java
Integer num = 10; 
Integer newNum = num + 5; // Новый объект
```
    
3. **Классы из библиотеки `java.time` (даты и времени):** [[Работа с датой в java]]
    - `LocalDate`
    - `LocalTime`
    - `LocalDateTime`
    - `ZonedDateTime`
```java
LocalDate date = LocalDate.now();
LocalDate newDate = date.plusDays(5); // Создан новый объект
```
    
4. **Коллекции (неизменяемые обёртки):**
    - `Collections.unmodifiableList(List)`
    - `Collections.unmodifiableMap(Map)`
    - `List.of(...)` (Java 9+)
    - `Set.of(...)` (Java 9+)
    - `Map.of(...)` (Java 9+)
```java
List<String> list = List.of("A", "B", "C");
list.add("D"); // UnsupportedOperationException
```
    
5. **Кастомные неизменяемые классы:**
    - Классы, созданные с использованием принципов неизменяемости (например, `final` поля, отсутствие сеттеров).

#### **Особенности неизменяемых объектов:**
- Не содержат методов, которые изменяют внутреннее состояние.
- Все поля должны быть объявлены как `final` и инициализированы один раз (в конструкторе).
- ==**Гарантируют потокобезопасность, но создают накладные расходы при изменении==**
- Используются в качестве ключей в коллекциях (`HashMap`, `HashSet`).

---
#### **Как создать неизменяемый объект**

Для создания собственного неизменяемого класса следуйте следующим принципам:
1. **Сделать класс `final`**, чтобы никто не мог его наследовать.
2. **Объявить все поля `private` и `final`**.
3. **Инициализировать поля только через конструктор**.
4. **Не предоставлять сеттеры**.
5. **Если объект содержит изменяемые объекты, вернуть их копии (а не оригинал)**.

*Пример неизменяемого класса*:
```java
final class ImmutablePerson {
    private final String name;
    private final int age;

    // Конструктор
    public ImmutablePerson(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Геттеры
    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    // Метод toString для удобства
    @Override
    public String toString() {
        return "ImmutablePerson{name='" + name + "', age=" + age + "}";
    }
}
```

## **Ключевые моменты**
- Используйте изменяемые объекты, если данные будут часто обновляться.
- Используйте неизменяемые объекты, если данные фиксированы или нужна потокобезопасность.
- В большинстве приложений используется сочетание обоих типов объектов для достижения баланса между производительностью и безопасностью.


[^2]:### Объяснение метода с `Collections.unmodifiableList`
	
	Когда класс содержит изменяемые объекты, такие как списки, важно предотвратить возможность их изменения извне. В данном случае поле `subjects` является изменяемым объектом (`List<String>`), и если мы вернём его напрямую, то внешний код сможет изменить содержимое списка, нарушив неизменяемость класса.
	### **Почему нужно защищать изменяемые объекты**
	Рассмотрим проблему:
	
	```java
	ImmutableStudent student = new ImmutableStudent("John", Arrays.asList("Math", "Physics"));
	
	// Если метод возвращает сам список (а не копию), внешний код может его изменить:
	List<String> subjects = student.getSubjects();
	subjects.add("Chemistry"); // Это изменит внутренний список объекта student!
	```
	Здесь внешний код получил ссылку на внутренний список `subjects` и изменил его содержимое, что нарушает концепцию неизменяемости.
	
	### **Как это решается**
	Вместо возврата ссылки на исходный список, метод `getSubjects` возвращает **неизменяемую копию списка**, созданную с помощью `Collections.unmodifiableList`. Это гарантирует, что попытка изменить список вызовет исключение `UnsupportedOperationException`.
	
	**Пример:**
	```java
	public List<String> getSubjects() {
	return Collections.unmodifiableList(subjects);
	}
	```
	
	- `Collections.unmodifiableList(subjects)` возвращает "обёртку" над оригинальным списком. Эта обёртка блокирует любые попытки изменения списка.
	- Если внешний код попробует, например, добавить элемент, то выбросится исключение.
	
	---
	### **Как это работает на практике**
	Пример использования:
	```java
	import java.util.Arrays;
	
	public class Main {
	    public static void main(String[] args) {
	        ImmutableStudent student = new ImmutableStudent("John", Arrays.asList("Math", "Physics"));
	
	        // Получаем неизменяемый список
	        List<String> subjects = student.getSubjects();
	
	        System.out.println("Subjects: " + subjects);
	
	        // Попытка изменить список вызовет исключение
	        try {
	            subjects.add("Chemistry");
	        } catch (UnsupportedOperationException e) {
	            System.out.println("Attempt to modify subjects failed: " + e);
	        }
	    }
	}
	```
	***Результат выполнения:***
	```yaml
	Subjects: [Math, Physics]
	Attempt to modify subjects failed: java.lang.UnsupportedOperationException
	```
	
	### **Ключевые моменты:**
	1. **Безопасность**: Внешний код не может изменить внутреннее состояние объекта.
	2. **Исключение при изменении**: Если кто-то попытается изменить список, это сразу же вызовет ошибку, позволяя отловить проблему.
	3. **Простота реализации**: Метод легко добавить в класс, где требуется защита изменяемых данных.
	
	Таким образом, этот подход помогает сохранить неизменяемость объекта, даже если он содержит изменяемые типы данных.
