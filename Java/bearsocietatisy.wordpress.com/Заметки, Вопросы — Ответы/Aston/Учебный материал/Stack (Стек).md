Стек в Java представляет собой область памяти, используемую для хранения вызовов методов, локальных переменных и временных данных. Каждая нить выполнения (поток) получает свой отдельный стек, что позволяет изолировать выполнение методов между потоками.

![[PROgrammer-Stack_1.gif|500]]
> Что такое **стек**? Стековая память в Java работает по схеме LIFO (Последний-зашел-Первый-вышел). Всякий раз, когда вызывается метод, в памяти стека создается новый блок, который содержит примитивы и ссылки на другие объекты в методе. Как только метод заканчивает работу, блок также перестает использоваться, тем самым предоставляя доступ для следующего метода.
> 
> Размер стека намного меньше объема памяти в куче.
> 
> Всякий раз, когда создается объект, он всегда хранится в куче, а в памяти стека содержится ссылка на него. Память стека содержит только локальные переменные примитивных типов и ссылки на объекты в куче.
### **Состав стека**
1. **Фреймы стека (Stack Frames)**  
    Стек состоит из фреймов, каждый из которых создаётся при вызове метода и уничтожается после завершения метода.  
    **Фрейм стека включает:**
    - **Локальные переменные:**
        - Переменные, объявленные внутри метода, включая параметры метода.
        - Примитивные типы данных (`int`, `double`, `char` и т. д.) и ссылки на объекты.
    - **Операнды (Operand Stack):**
        - Используется для временного хранения данных во время вычислений.
        - Например, промежуточные результаты математических операций.
    - **Метаданные метода:**
        - Адрес возврата (где продолжить выполнение после завершения метода).
        - Информация о методе (например, его идентификатор).
2. **Локальные переменные** Хранят значения, которые используются только внутри метода.
	*Пример:*
```java
void method(int x) {
    int y = 5; // Локальная переменная
    int z = x + y; // z также локальная переменная
}
```
`x`, `y`, `z` находятся в стеке, потому что они принадлежат методу.
3. **Аргументы метода** При вызове метода его параметры копируются в стек.  
	*Пример:*
```java
void add(int a, int b) {
    int sum = a + b; // a, b и sum находятся в стеке
}
```
4. **Адрес возврата**
	- Указывает место в коде, куда нужно вернуться после завершения метода.
	- Если метод вызывает другой метод, текущий адрес сохраняется в фрейме стека.
1. **Операнды**
    - Временные данные, участвующие в вычислениях.
    - Например, для операции `a + b` операнды `a` и `b` помещаются в стек, а затем результат вычисления.
