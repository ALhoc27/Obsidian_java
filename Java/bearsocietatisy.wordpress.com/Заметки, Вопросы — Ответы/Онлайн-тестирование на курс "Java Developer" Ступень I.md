### Что такое абстрактный класс?

Абстрактный класс в Java — это класс, который не может быть непосредственно инстанцирован (создан объект). Он используется как базовый класс для других классов. Абстрактные классы определяются с использованием ключевого слова `abstract`.
#### Особенности абстрактного класса:

1. **Может содержать абстрактные методы:**
    - Абстрактный метод — это метод без реализации, только с объявлением (без тела метода). Подклассы должны его реализовать.

```java
abstract class Shape {
    abstract void draw(); // Абстрактный метод
}
```

- **Может содержать неабстрактные методы:**
    - Абстрактный класс может иметь методы с реализацией, которые подклассы могут использовать или переопределять.

- **Может иметь поля, конструкторы и статические методы:**
    - Конструкторы в абстрактных классах используются для инициализации общих данных, которые будут наследоваться подклассами.

- **Наследование:**
    - Классы, наследующие абстрактный класс, должны реализовать все его абстрактные методы, иначе они тоже должны быть объявлены абстрактными.

- **Не может быть инстанцирован:**
    - Попытка создать объект абстрактного класса вызовет ошибку компиляции:
```java
Shape shape = new Shape(); // Ошибка
```

##### Методы, которые не могут быть переопределены:

1. **`private` методы:**
    - **Почему не могут быть переопределены:** Методы с модификатором `private` **не доступны** в наследующих классах, потому что они ограничены только внутри своего класса. Поскольку доступ к таким методам невозможно получить в подклассе, их нельзя переопределить.

```java
class Parent {
    private void doSomething() { }
}

class Child extends Parent {
    @Override
    private void doSomething() { } // Ошибка: нельзя переопределить private метод
}
```

2. **`static` методы:**
	 - **Почему не могут быть переопределены:** `static` методы принадлежат классу, а не экземпляру класса. Переопределение в классах-наследниках происходит только для экземплярных методов, то есть тех, которые могут быть вызваны через объект. Для `static` методов работает механизмы **скрытия** методов (method hiding), а не переопределение.
[[Учебный материал#Сокрытие методов]]
```java
class Parent {
    public static void doSomething() { }
}

class Child extends Parent {
    @Override
    public static void doSomething() { } // Ошибка: нельзя переопределить static метод
}
```

3. **`final` методы:**
	- **Почему не могут быть переопределены:** Методы с модификатором `final` не могут быть переопределены, поскольку `final` означает, что метод не может быть изменен (переопределен) в подклассах.
```java
class Parent {
    public final void doSomething() { }
}

class Child extends Parent {
    @Override
    public final void doSomething() { } // Ошибка: нельзя переопределить final метод
}
```

Цикл `do...while` — это один из базовых циклов в программировании, который гарантирует выполнение тела цикла хотя бы один раз, даже если условие сразу окажется ложным. Это отличие от цикла `while`, который сначала проверяет условие, а затем выполняет тело.
#### Синтаксис

```java
do {
    // Тело цикла — выполняется хотя бы один раз
} while (условие);
```

#### Особенности
1. **Обязательное выполнение тела цикла хотя бы один раз**: Условие проверяется только после выполнения тела.
2. **Типичная область применения**: Когда действия в цикле нужно выполнить хотя бы один раз, например, для чтения ввода пользователя, предложив повторить ввод при неверных данных.

`switch` — это конструкция управления в Java, которая позволяет выполнять различные блоки кода в зависимости от значения выражения. Это удобная альтернатива множественным `if-else` конструкциям.

#### Синтаксис `switch`
```java
switch (выражение) {
    case значение1:
        // Блок кода для случая значение1
        break;
    case значение2:
        // Блок кода для случая значение2
        break;
    ...
    default:
        // Блок кода, если ни один из случаев не подходит
}
```

#### Компоненты

1. **`выражение`**:
    - Должно быть значением типа `byte`, `short`, `char`, `int`, `String`, или `enum` (начиная с Java 7).
    - Начиная с Java 14, поддерживаются выражения типа `var`, если они определяют одно из перечисленных значений.

1. **`case`**:
    - Задает возможные значения, с которыми сравнивается результат `выражения`.
    - Каждое значение должно быть уникальным и неизменяемым.

1. **`break`**:
    - Завершает выполнение блока `switch`.
    - Если его не указать, выполнение продолжится с следующего `case` (так называемое "проваливание" или _fall-through_).

1. **`default`**:
    - Указывается для обработки всех случаев, которые не подходят под определенные `case`.
    - Необязателен.

Правильные способы создания массива в Java:
1. **`int numbers[] = new int[10];`**  
    Это корректный способ объявления и инициализации массива с типом `int` и размером 10.
    
2. **`int[] numbers = new int[10];`**  
    Это также правильный способ создания массива с типом `int` и размером 10, но скобки ставятся перед именем переменной. (**Лучше именно так объявлять массивы**)

Ошибочные способы:
- **`int[] numbers = new int(10);`** — Это ошибка. Для создания массива используется синтаксис `new int[10]`, а не `new int(10)`.
- **`int[] numbers = new array[10];`** — Это ошибка. Массив должен быть создан с типом `int[]`, а не с типом `array`. Правильный синтаксис: `int[] numbers = new int[10];`. [^6]
- **`int numbers[] = new array[10];`** — Это ошибка. Массив должен быть типа `int`, а не типа `array`. Правильный синтаксис: `int[] numbers = new int[10];`.
- **`int numbers[] = new inti(10);`** — Это ошибка. `new inti(10)` не существует. Правильный синтаксис для создания массива — `new int[10]`.

Метод с таким определением:
```java
public static int method(int[] array, int... vararg) {
    return 1;
}
```

Этот метод принимает два параметра:
1. **`int[] array`** — массив целых чисел.
2. **`int... vararg`** — переменное количество аргументов типа `int`. Это называется **varargs**, и позволяет передавать любое количество аргументов типа `int` (или не передавать их вообще).

Теперь давайте рассмотрим каждый способ вызова метода:
### Правильные способы вызова:

1. **`method(1, 2, 3);`** — **Ошибка**.
    - Метод ожидает первый параметр как массив, но здесь переданы просто значения. Нужно передать массив, например: `new int[]{1, 2, 3}`.

1. **`method(new int[] {1, 2, 3}, 4, 5);`** — **Правильный**.
    - Здесь передается массив `new int[] {1, 2, 3}` как первый параметр и два дополнительных значения `4` и `5` для varargs. Это правильный синтаксис.

1. **`method(new int[] {1, 2, 3});`** — **Правильный**.
    - Здесь передается массив `new int[] {1, 2, 3}` как первый параметр и нет дополнительных значений для varargs. Это также правильный синтаксис.

1. ==**`method(new int[] {1, 2, 3}, new int[] {4, 5, 6});`** — **Ошибка**.==
    - ==Второй параметр передается как массив `new int[] {4, 5, 6}`, но он должен быть передан как переменное количество аргументов (varargs), а не как массив. Правильный способ — передать элементы через запятую, а не массив. Правильный вызов был бы таким: `method(new int[] {1, 2, 3}, 4, 5, 6);`.== [^7]

1. **`method();`** — **Ошибка**.
    - Метод требует хотя бы один массив `int[]`, поэтому вызов без параметров является некорректным.

[^4]:В Java статические методы нельзя переопределить в строгом смысле этого слова, так как они связаны с классом, а не с его экземпляром. Однако их можно **скрыть** в подклассе, определив статический метод с тем же именем и сигнатурой.
	###### Как работает скрытие статических методов
	
	Если вы создадите статический метод с таким же именем и параметрами в подклассе, то метод суперкласса будет скрыт. Вызываемый метод определяется в зависимости от того, откуда осуществляется вызов (из суперкласса или подкласса).
	
	###### Пример скрытия статического метода
	
	```java
	class SuperClass {
	    public static void staticMethod() {
	        System.out.println("Static method in SuperClass");
	    }
	}
	
	class SubClass extends SuperClass {
	    public static void staticMethod() {
	        System.out.println("Static method in SubClass");
	    }
	}
	
	public class Main {
	    public static void main(String[] args) {
	        SuperClass.staticMethod(); // Выведет: Static method in SuperClass
	        SubClass.staticMethod();  // Выведет: Static method in SubClass
	
	        // Но:
	        SuperClass obj = new SubClass();
	        obj.staticMethod(); // Выведет: Static method in SuperClass
	    }
	}
	```
	
	Доступность всегда определяется **типом ссылки**, а не от типом объекта.
	
	> [!NOTE]
	> Скрытие статических методов — это не переопределение, а механизм, связанный с типами классов. Вызов всегда разрешается на уровне компиляции.
