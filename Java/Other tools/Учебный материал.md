### Логические операторы в Java

1. **Логическое И (`&&`):**
    - Возвращает `true`, если оба операнда истинны.
    - Если первый операнд ложен, второй операнд **не вычисляется** (короткое замыкание).
    - Пример:
```java
boolean result = (5 > 3) && (4 > 2); // true
boolean result2 = (5 > 3) && (4 < 2); // false
```

1. **Логическое ИЛИ (`||`):**
    - Возвращает `true`, если хотя бы один операнд истинен.
    - Если первый операнд истинен, второй операнд **не вычисляется** (короткое замыкание).
    - Пример:
```java
boolean result = (5 > 3) || (4 < 2); // true
boolean result2 = (5 < 3) || (4 < 2); // false
```

1. **Логическое НЕ (`!`):**
    - Инвертирует логическое значение.
    - Если выражение истинно, становится ложным, и наоборот.
    - Пример:
```java
boolean result = !(5 > 3); // false
```
        
4. **Побитовое И (`&`):**
    - Работает как логическое И, но **всегда вычисляет оба операнда**, даже если первый уже ложен.
    - Пример:
```java
boolean result = (5 > 3) & (4 < 2); // false
```

5. **Побитовое ИЛИ (`|`):**
    - Работает как логическое ИЛИ, но **всегда вычисляет оба операнда**, даже если первый уже истинен.
    - Пример:
```java
boolean result = (5 > 3) | (4 < 2); // true
```
        
6. **Исключающее ИЛИ (`^`):**
    - Возвращает `true`, если один из операндов истинен, но не оба.
    - Пример:
```java
boolean result = (5 > 3) ^ (4 < 2); // true
boolean result2 = (5 > 3) ^ (4 > 2); // false
```

#### Особенности:

1. **Короткое замыкание:**
    - `&&` и `||` прекращают вычисление, если результат уже известен:
        - `&&` останавливается, если первый операнд равен `false`.
        - `||` останавливается, если первый операнд равен `true`.

1. **Побитовые операторы (`&`, `|`, `^`):**
    - Применяются также к числовым типам (`int`, `long`) для операций на уровне битов.

1. **Приоритет операторов:**
    - Логическое НЕ (`!`) имеет самый высокий приоритет.
    - Затем идут `&&` и `||` в порядке их появления.

### Модификаторы доступа

- **`private`** — доступ только внутри класса.
- **`default` (package-private)** — доступ внутри того же пакета.
- **`protected`** — доступ внутри того же пакета и в подклассах.
- **`public`** — доступ из любого места.


### Сокрытие методов
**Скрытие методов** (method hiding) в Java происходит, когда метод в **подклассе** имеет такую же сигнатуру, как и метод в **родительском классе**, но это происходит для **`static` методов** ^[`static` методы принадлежат классу, а не экземпляру класса. Переопределение в классах-наследниках происходит только для экземплярных методов, то есть тех, которые могут быть вызваны через объект. Для `static` методов работает механизмы **скрытия** методов (method hiding), а не переопределение.]. В отличие от **переопределения методов**, где используется механизм полиморфизма для динамического выбора метода в зависимости от типа объекта, скрытие методов касается **`static` методов** и является примером статического связывания.

###### Важные особенности скрытия методов:
1. **`static` методы** в родительском классе могут быть скрыты в подклассе методом с той же сигнатурой (именем и параметрами).
2. В отличие от **переопределения**, где метод вызывается на основе объекта, для **скрытых методов** выбор метода зависит от типа ссылки, а не от объекта.
3. Ключевое отличие скрытия методов в том, что это **не переопределение**, так как `static` методы привязаны к классу, а не к объекту.

```java
class Parent {
    public static void doSomething() {
        System.out.println("Parent static method");
    }
}

class Child extends Parent {
    // Скрытие метода
    public static void doSomething() {
        System.out.println("Child static method");
    }
}

public class Main {
    public static void main(String[] args) {
        Parent parent = new Parent();
        Parent childAsParent = new Child();
        Child child = new Child();

        parent.doSomething();         // Вывод: Parent static method
        childAsParent.doSomething();  // Вывод: Parent static method
        child.doSomething();          // Вывод: Child static method
    }
}
```
###### Разбор:
1. **В родительском классе** `Parent` определен `static` метод `doSomething()`, который выводит строку "Parent static method".
2. **В подклассе** `Child` скрыт этот метод, так как в нем определен `static` метод с той же сигнатурой `doSomething()`, который выводит строку "Child static method".
3. При вызове метода:
    - Если метод вызывается через объект типа `Parent`, будет вызван метод родительского класса (`Parent static method`).
    - Если метод вызывается через объект типа `Child`, будет вызван метод подкласса (`Child static method`), но только при прямом вызове через `Child`.

###### Ключевые моменты:
- Скрытие происходит **только для `static` методов**. Это связано с тем, что `static` методы привязываются к классу, а не к экземпляру объекта.
- При вызове метода через **ссылку на родительский класс** будет вызываться метод родительского класса, даже если объект на самом деле является экземпляром подкласса. Это пример статического связывания.
- Ссылки на **подклассы** вызовут метод подкласса, если метод скрыт, но это не связано с полиморфизмом (как при переопределении методов).

###### Сравнение с переопределением:

1. **Переопределение (overriding)** — это механизм, когда метод родительского класса заменяется методом подкласса при работе с объектами.
    - Работает для **не-`static` методов**.
    - Метод выбирается в момент выполнения (динамическое связывание).

1. **Скрытие метода (method hiding)** — это когда метод родительского класса скрывается методом в подклассе.
    - Работает только для **`static` методов**.
    - Метод выбирается на основе типа ссылки (статическое связывание).

###### Итог:
- **Скрытие метода** происходит только для `static` методов, и метод выбирается в зависимости от типа ссылки.
- **Переопределение метода** работает для экземплярных методов и выбирается на основе типа объекта (динамическое связывание).


### **Перегруженные методы (Method Overloading)**

Перегрузка методов — это механизм в Java, который позволяет в одном классе определять **несколько методов с одинаковым именем**, но с **разными параметрами**. Эти методы могут отличаться количеством, типом или порядком параметров. Перегрузка методов не зависит от модификаторов доступа или возвращаемого типа.
###### **Особенности перегрузки методов:**
1. **Разные параметры:**
    - Перегруженные методы должны иметь **различия в сигнатуре** (количестве или типе параметров). Это позволяет компилятору определить, какой именно метод нужно вызвать.
    - Если сигнатуры методов одинаковы, компилятор выдаст ошибку.

2. **Независимость от возвращаемого типа:**
    - Изменение возвращаемого типа не считается перегрузкой, если сигнатура остается одинаковой.
    - **Пример:**
```java
void doSomething(int a); 
int doSomething(int a); // Ошибка! Это не перегрузка.
```
        
3. **Модификаторы доступа:**
    - Методы могут иметь разные модификаторы (`public`, `protected`, `private`), но это не влияет на перегрузку.
    
4. **Наследование:**
    - Перегрузка методов может происходить как в одном классе, так и между классом-родителем и подклассом.


###### Пример перегрузки методов:
```java
class OverloadExample {
    // Метод с одним параметром
    public void doSomething(int a) {
        System.out.println("Method with int: " + a);
    }

    // Метод с двумя параметрами
    public void doSomething(int a, int b) {
        System.out.println("Method with two ints: " + (a + b));
    }

    // Метод с другим типом параметра
    public void doSomething(String s) {
        System.out.println("Method with String: " + s);
    }
}

public class Main {
    public static void main(String[] args) {
        OverloadExample obj = new OverloadExample();
        obj.doSomething(5);             // Вызов метода с int
        obj.doSomething(5, 10);         // Вызов метода с двумя int
        obj.doSomething("Hello");       // Вызов метода с String
    }
}
```

###### Результат:
```java
Method with int: 5
Method with two ints: 15
Method with String: Hello
```

###### **Частые вопросы о перегрузке:**

1. **Можно ли перегрузить методы, изменив только возвращаемый тип?**  
    Нет. Перегрузка основана на различиях в параметрах, а не в возвращаемом типе.
    
2. **Можно ли перегрузить `static` методы?**  
    Да. `static` методы могут быть перегружены, так как перегрузка не связана с экземплярами класса.
    
3. **Можно ли перегрузить методы в подклассе?**  
    Да. Подкласс может добавить свои версии методов с новой сигнатурой, и это будет считаться перегрузкой.
    
4. **Перегрузка работает с разными модификаторами?**  
    Да, модификаторы доступа не влияют на перегрузку.

###### Итог:
**Перегрузка методов** позволяет повысить читаемость и гибкость кода, предоставляя возможность использовать одно имя метода для выполнения схожих операций с разными типами или количеством параметров.


### **Varargs (переменные аргументы)**
**Varargs (переменные аргументы)** в Java — это способ передачи переменного числа аргументов в метод, когда точное количество передаваемых значений заранее неизвестно. Это позволяет сделать код более гибким и удобным.

Чтобы использовать varargs, нужно в методе указать тип переменной с троеточием `...` перед именем параметра. Это позволяет передавать в метод любое количество аргументов указанного типа.

###### Пример синтаксиса:
```java
public static void method(int... numbers) {
    // метод может принимать любое количество аргументов типа int
}
```

> Здесь параметр `numbers` будет массивом типа `int[]`. Вы можете передать любое количество значений типа `int` или даже не передавать аргументы вообще (в этом случае будет пустой массив).

##### Важные моменты:

1. **varargs должен быть последним параметром**:
    - Если метод принимает несколько параметров, то **varargs должен быть последним**.[^1] Нельзя поместить после него другие аргументы.

2. **Varargs является массивом**:
    - Внутри метода параметр `varargs` фактически является массивом. То есть вы можете обращаться к переданным значениям как к элементам массива.
    
3. **Передача фиксированного числа аргументов**:
    - Вы можете передавать: несколько значений, **а также передавать массив как один аргумент** (==но это не совсем корректно==: **IDEA подчеркнет что это избыточно**), или не передавать вообще не чего.
```java
printNumbers(new int[]{1, 2, 3});  // Компилятор пропустит но правильная передача аргументов по значениям (IDEA подчеркнет что это избыточно)
```
**правильно:**
```java
printNumbers(1, 2, 3);  // Правильно
```
